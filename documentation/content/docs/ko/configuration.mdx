---
title: 설정 옵션
description: DariConfig 옵션 레퍼런스
---

`Dari.initialize()`에 `DariConfig` 인스턴스를 전달하여 동작을 커스터마이징합니다.

```kotlin
Dari.initialize(
    context = this,
    config = DariConfig(
        maxEntries = 500,
        showNotification = true,
        maxContentLength = 500_000,
        shakeToOpen = false,
        retentionPeriod = 1.days,
        fireAndForget = false,
    )
)
```

## 옵션 설명

### `maxEntries`
**타입**: `Int` | **기본값**: `500`

메모리에 유지할 최대 메시지 수입니다. 초과 시 오래된 항목부터 삭제됩니다.

### `showNotification`
**타입**: `Boolean` | **기본값**: `true`

상태 알림을 표시합니다. 알림을 탭하면 인스펙터가 열립니다.

### `maxContentLength`
**타입**: `Int` | **기본값**: `500_000`

요청/응답 본문 데이터의 최대 문자 수입니다. 초과 시 잘립니다.

### `shakeToOpen`
**타입**: `Boolean` | **기본값**: `false`

`true`로 설정하면 디바이스를 흔들어 Dari 인스펙터를 열 수 있습니다.

### `retentionPeriod`
**타입**: `Duration?` | **기본값**: `null`

TTL 기반 자동 정리입니다. 이 기간보다 오래된 메시지는 자동으로 삭제됩니다.

```kotlin
DariConfig(retentionPeriod = 6.hours)
```

`null`로 설정하면 TTL 정리가 비활성화됩니다.

### `fireAndForget`
**타입**: `Boolean` | **기본값**: `false`

`true`로 설정하면 모든 브릿지 호출이 응답을 기다리지 않고 즉시 `SUCCESS`로 처리됩니다. 전체 브릿지가 단방향인 경우 유용합니다.

개별 호출에 적용하려면 인터셉터 메서드의 `fireAndForget` 파라미터를 사용하세요.
